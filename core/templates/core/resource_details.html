Basics of PIP

What is PIP & Why It Is Needed?

  • Python comes as standard distribution, but Python has a huge community, and every package cannot be given in standard distribution; otherwise, its size would be increased.
  • Hence, Python provides a tool named PIP (Package Installer for Python), using which we can install and manage additional packages and libraries. So, PIP is a package manager in Python.
  • While working on projects, we often need additional packages; thus, it is an essential tool for developers.

How to Install PIP?

  • PIP has been included with the Python installer since version 3.4 unless you are using an older version.
  • To use PIP, you need to install Python, which comes with PIP included.

How to Verify PIP is Installed or Not?

By running the following commands:

  • pip --version: This command shows the version of PIP installed, along with the Python version it's associated with and the location of the PIP installation.
  • pip -V: This is shorthand for pip --version and gives the same output.
  • py -m pip --version: This command tells Python to run the PIP module using the Python executable (py) installed on your system. It shows the PIP version, the associated Python version, and the location where PIP is installed. This command is particularly useful if you have multiple versions of Python installed.

Note: The key difference is that py -m pip --version explicitly invokes PIP through the py launcher, ensuring that PIP runs within the context of a specific Python version.

Comprehensive PIP Commands List

  • pip install package_name – Used to install a package.
  • pip install --upgrade package_name – Used to upgrade an installed package.
  • pip uninstall package_name – Used to uninstall a package.
  • pip list – Used to list installed packages.
  • pip show package_name – Used to show details about a specific package.
  • pip freeze > requirements.txt – Used to freeze installed packages to a requirements file.
  • pip install -r requirements.txt – Used to install packages from a requirements file.
  • pip list --outdated – Used to check for outdated packages.
  • pip check – Used to verify installed packages have compatible dependencies.
  • pip search search_term – Used to search for packages by name.
  • pip install --no-cache-dir package_name – Used to install a package without using the cache.
  • pip install --user package_name – Used to install a package for the current user only.
  • pip wheel package_name – Used to create a wheel distribution of a package.
  • pip uninstall -y package_name – Used to uninstall a package without prompting for confirmation.

Installing Packages with PIP

Whenever we use the PIP command to install a package, it can be sourced from two main places:

  1. Online Repositories: (PyPI & GitHub)

    To install packages from PyPI, you can use the command pip install <package_name>. Make sure you are connected to the internet.

  2. Distribution Files: Packages can be installed from local distribution files (tar files, wheel files, etc.) stored on your computer.
    • To install a Wheel (.whl) File: pip install path/to/your-package.whl
    • To install a Source Distribution (.tar.gz) File: pip install path/to/your-package.tar.gz

How to Check Dependency of One Package on Another

To check if a package depends on another, use the command:

pip show package_name to find the dependencies of the specified package. Look for Requires in the output.
